> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

| Property | Value |

## Spacing Scale

Declared values (all multiples of 4, aligned with Tailwind's default scale as used in Phase 1):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip internal padding (px-1 py-0.5), badge padding |
| sm | 8px | Compact element spacing, card inner padding (p-2), chip gap (gap-2) |
| md | 16px | Default element spacing, page horizontal padding (px-4), form field gap |
| lg | 24px | Section padding (p-6), heading bottom margin (mb-6), page top padding (py-6) |
| xl | 32px | Layout vertical gaps, section breaks (mt-8) |
| 2xl | 48px | Empty/loading state vertical padding (py-20 equivalent areas) |
| 3xl | 64px | Page-level top spacing for centered forms (mt-16, mt-20) |

Exceptions:
- Chip toggle buttons (genre + mood): minimum touch target 44px height enforced via `min-h-[44px]` on mobile viewports (accessibility — WCAG 2.5.5)
- Navbar: fixed height `py-3` (12px top/bottom) — preserve existing Navbar.tsx spacing exactly
- Explanation text below MovieCard: `mt-1` (4px) gap between card bottom edge and explanation sentence

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label | 14px | 500 (medium) | 1.4 | `text-sm font-medium` |
| Heading | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |
| Supporting/meta | 12px | 400 (regular) | 1.4 | `text-xs` |

Source: extracted from `MovieCard.tsx` (text-sm, text-xs, font-semibold), `HomePage.tsx` (text-2xl font-bold), `LoginPage.tsx` (text-sm font-medium), `Navbar.tsx` (text-xl font-bold for brand, text-sm for nav items).

**Phase 2 specific typography rules:**
- Section label above chip groups (e.g., "Select Genres", "How are you feeling?"): `text-sm font-medium text-gray-700` — same as form field labels in LoginPage
- Chip text: `text-sm font-medium` — same weight as labels for legibility at small size
- Explanation text below MovieCard: `text-xs text-gray-500` — 12px, weight 400 — clearly subordinate to card title
- "Optional" label next to mood section heading: `text-xs text-gray-400 font-normal` inline after heading
- Preference section heading on `/recommendations` page: `text-2xl font-bold text-gray-900` — consistent with HomePage h1

## Color

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | #f9fafb | `gray-50` | Page background (`bg-gray-50`), outer wrapper fill |
| Secondary (30%) | #ffffff | `white` | Cards (`bg-white`), preference form surface, chip unselected state background |
| Nav surface | #111827 | `gray-900` | Fixed top navbar (existing — do not change) |
| Accent (10%) | #2563eb | `blue-600` | Chip selected state, primary CTA button, focus rings (`focus:ring-blue-500`), "For You" nav link active state |
| Destructive | #dc2626 | `red-600` | Validation error text only (e.g., "Select at least 1 genre") |

Accent reserved for:
1. Selected genre chip background (`bg-blue-600 text-white`)
2. Selected mood chip background (`bg-blue-600 text-white`)
3. "Get My Recommendations" CTA button (`bg-blue-600 hover:bg-blue-700`)
4. "For You" navbar link when on `/recommendations` route (`text-blue-400` — consistent with existing Navbar hover state)
5. Focus rings on all interactive elements (`focus:ring-2 focus:ring-blue-500`)

Chip unselected state: `bg-white border border-gray-300 text-gray-700 hover:border-blue-400 hover:text-blue-600` — provides clear selected vs. unselected affordance without inventing new tokens.

Mood chip distinction from genre chips: same visual style (chip shape, border, color tokens) but rendered under a separate labeled section ("How are you feeling?" heading). No color difference — visual separation is achieved by grouping and heading label, not color change. Source: CONTEXT.md decision "same visual style as genre chips".

Rating badge on MovieCard (existing): `bg-yellow-400 text-yellow-900` — preserve exactly as-is.

## Component Inventory

New components introduced in Phase 2 and their visual specifications:

### PreferenceChip (genre and mood chips)

```
Unselected: bg-white border border-gray-300 rounded-full px-4 py-2 text-sm font-medium text-gray-700
             hover:border-blue-400 hover:text-blue-600 transition-colors cursor-pointer
```

Genre chips: multi-select (multiple can be selected simultaneously).
Mood chips: single-select (selecting one deselects the previous; radio behavior).

### PreferenceForm (the onboarding/edit form surface)

```

Genre group: flex flex-wrap gap-2 mt-2

             disabled:opacity-50 transition-colors (consistent with LoginPage submit button)
```

Validation error: `text-sm text-red-600 mt-1` below the genre group when no genre is selected and user tries to submit.

### EditPreferencesControl (collapsible section above results grid)

```
Trigger button: px-4 py-2 text-sm font-medium bg-white border border-gray-300 rounded-lg
                hover:bg-gray-50 transition-colors (consistent with pagination buttons in HomePage)
Expanded state: PreferenceForm renders below trigger button with mt-4 spacing
Collapsed state: form hidden, only trigger button visible
Transition: no animation required — show/hide via React state only
```

### RecommendationCard (MovieCard wrapper with explanation)

```
Wrapper:       flex flex-col
MovieCard:     existing MovieCard component — no modifications
Explanation:   mt-1 text-xs text-gray-500 px-0 (aligns with card edge, no additional horizontal padding)
Explanation copy format: "Recommended because you like [Genre1] and [Genre2]"
```

### RecommendationsPage layout

```
Container:  max-w-7xl mx-auto px-4 py-6 (consistent with HomePage)
Page h1:    text-2xl font-bold text-gray-900 mb-4

State A — no preferences saved:
  Shows PreferenceForm centered, no grid below.
  No empty-state illustration needed.

State B — preferences saved, loading:
  EditPreferencesControl (collapsed) at top
  MovieGrid skeleton (8 cards, animate-pulse) below — reuse existing SkeletonCard

State C — preferences saved, results loaded:
  EditPreferencesControl (collapsed) at top
  Grid of RecommendationCards (2/3/4/5 columns responsive — same breakpoints as MovieGrid)

State D — error from API:
  EditPreferencesControl (collapsed) at top
  Error banner below (see Copywriting Contract)
```

## Interaction Contract

### Chip selection behavior

Genre chips (multi-select):
- Click/tap to toggle selected state
- At least 1 must be selected before CTA is enabled
- CTA is `disabled:opacity-50` and non-interactive when 0 genres selected
- No maximum genre count enforced

Mood chips (single-select / radio):
- Click/tap to select; clicking the currently-selected mood chip deselects it (toggles off)
- At most 1 selected at any time
- Entirely optional — form is valid with 0 moods selected

### Preferences persistence UX

- On first visit to `/recommendations`: PreferenceForm shown (State A)
- After submitting preferences: form collapses, results load (State B → C)
- Preferences stored in backend (`user_preferences` collection); page re-mounts with existing preferences pre-selected
- "Edit Preferences" button expands form with current selections pre-populated
- "Update Recommendations" re-submits and triggers new fetch; results update in place

### Navigation

- "For You" link added to Navbar.tsx between brand link and auth section
- Visible only when `isAuthenticated === true` (follows existing Navbar pattern)
- Link: `<Link to="/recommendations">` with class `text-sm hover:text-blue-400 transition-colors`
- Active state (when on `/recommendations`): `text-blue-400` (consistent with existing hover color — no NavLink active class needed for MVP)

### Loading and error states

- Loading: reuse `MovieGrid` skeleton (8 `SkeletonCard` placeholders, `animate-pulse`)
- Error: banner card `bg-red-50 border border-red-200 rounded-lg p-4` with error copy + "Try Again" button (`text-sm text-red-700`)
- Refetch on "Try Again": calls `refetch()` from `useRecommendations` React Query hook

## Accessibility Notes

- All chip buttons must have `type="button"` to prevent form submission
- Selected chips must have `aria-pressed="true"` (toggle button pattern per ARIA 1.2)
- Mood chip group must use `role="radiogroup"` with `aria-label="Mood selection (optional)"`
- Genre chip group must use `role="group"` with `aria-label="Genre selection"`
- Focus ring: `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-blue-500` on all chips and buttons
- Color contrast: `text-gray-700` on `bg-white` = 7.6:1 (WCAG AA compliant). `text-white` on `bg-blue-600` = 4.6:1 (WCAG AA compliant)
- Explanation text `text-xs text-gray-500` on `bg-white`: 4.6:1 — meets WCAG AA for normal text at 12px (large text threshold is 14px bold; 12px regular requires 4.5:1 minimum — passes)
